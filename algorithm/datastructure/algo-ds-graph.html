# 图的定义

一个图是 G = (V, E)是由顶点集V和边集E组成。每一条边就是一个顶点对(v, w)，v,w都是顶点集V中的顶点。边也被称作弧。如果顶点对是有序的就叫做有向图，顶点对是无序的就叫做无向图。树是一种特殊的无向图。

# 基本概念

度：从一个顶点发出的边数。

出度：有向图中，以一个顶点为起点的边数。

入度：有向图中，以一个顶点为终点的边数。

路径：路径是一个顶点序列，例如 ABC就是一个路径。

路径长度：路径中边的数量。

权重：与边相关的数，如距离。

环：如果一个顶点存在到其自身的路径，那么该路径构成一条环。

连通图：如果一个无向图中从每一个顶点到到其他顶点都存在一条路径，那么就称该图是连通的。

强连通图：如果一个有向图中从每一个顶点到其他顶点都存在一条路径，那么就称该图就强连通图。

弱连通图：将有向图的有向边替换为无向边，得到的基础图如果是连通的，则称该有向图为弱连通图。

极大连通子图：图G的一个子图，加入图中任何一个不存在该子图中的结点子图将不再连通，则称该子图为极大连通子图。

极小连通子图：图G的一个子图，删除该子图中任意一条边，子图将不再联通，称为该子图为极小连通子图。包含无向图G的所有顶点的极小连通子图就是最小生成树。

完全图：无向图任意两个顶点都存在边，叫做无向完全图。 有向图任意两个顶点都存在两个相反的有向边，叫做有向完全图。上边的两个示意图分别就是无向完全图和有向完全图。假如顶点数目为n， 那么完全图的边数是n(n-1)。

稀疏图：边的数目远远少于完全图。

稠密图：边的数目接近完全图。

# 图的存储结构

图的存储结构分为两种，一种是邻接矩阵，一种是邻接表。

如果一个图G(V, E)，V是顶点数，E是边数，那么用邻接矩阵表示图需要的空间为 |V|2, 如果是无向图，临近矩阵是个对称矩阵，优化后可以减少到 |V|2/2的空间。

无向图用邻接表表示需要的空间为 |V+2E|，有向图需要|V+E|。

# 无向图邻接矩阵和邻接表

如下图所示的无向图，分别给出了该无向图的邻接矩阵和邻接表的存储示意图。

若无向图中顶点个数为V, 邻接矩阵就是一个V行V列的矩阵。下面的无向图有4个顶点，邻接矩阵大小就是4x4。在邻接矩阵中，(vi，vj)表示从顶点 i 到 j 的边，若该边存在，则(vi，vj)=1，否则(vi，vj)=0。 如果该无向图带权，那么(vi，vj)=权重，权重为无穷表示该边不存在。 事实上，无向图邻接矩阵是一个对称矩阵，因为(vi，vj)和(vj，vi)是相同的边，所以无向图的存储只用存储图中阴影部分的边即可，优化后的需要的空间是|V|2/2。

图的另一种存储结构是邻接表。邻接表由两部分组成，顶点表和边表。其中顶点表是存储图中顶点的链表，边表是存储与该点相关的边的链表。
