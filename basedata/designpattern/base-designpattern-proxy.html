# 模式定义

代理模式是为其他对象提供一个代理对象以控制对这个对象的访问。 与装饰者模式的区别是，装饰者模式的目的是为了给原有对象添加额外的功能或行为， 代理模式侧重于不方便直接访问对象或者需要做一些访问控制。例如，我们可以使用代理模式来延迟实例化对象。

# 模式结构

  • 主题接口(Subject)：代理类和真实类的公共方法
  • 真实主题(RealSubject)：真实类，也就是被代理的类
  • 代理类(Proxy)

Proxy

public class Proxy implements Subject {
    private Subject subject;

    @Override
    public void request() {
        if (subject == null){
            System.out.println("Proxy初始化对象");
            subject = new RealSubject();
        }
        subject.request();
    }
}

# 优点缺点

优点：

  • 代理类能够对真实类做一些访问控制，封装了细节，降低了耦合，增加了灵活性和扩展性

缺点：

  • 在客户端和真实类之间增加了代理对象，可能会造成请求的处理速度变慢

# 使用时机

  • 远程代理，给存在于不同服务器上的资源提供一个本地的代理对象，如RFC调用
  • 虚拟代理，如果真实对象调用开销很大，可以先用一个代理创建一个小的对象代替真实对象
  • 安全代理，控制真实对象访问时的权限
  • 智能引用代理，当对象被调用时，做一些额外的操作，如检查是否被锁定，计算引用次数等
