# 模式定义

简单工厂模式又叫静态供工厂方法模式，可以通过传入的参数返回对应的类的实例。也就是说单独定义一个类实例化对象，其他类需要的时候只需传入一个参数即可，无须关注实例化细节。这个生产对象的类就叫做工厂类。 通常情况下，返回的实例具有相同的父类。 例如，利用简单工厂模式实现一个四则运算，我们只需要提供数字和运算符，就可以计算出结果。

# 模式结构

  • 抽象运算类(AbstractOperator)：抽象类，定义具体类的公共部分。 抽象类也可用接口替代。
  • 具体运算类(Operator)：具体类，定义具体逻辑实现
  • 工厂类(Factory)：生产对象的类

# 模式实现

抽象运算类：

public abstract class Operator {
    private double a;
    private double b;

    public double getA() {
        return a;
    }

    public void setA(double a) {
        this.a = a;
    }

    public double getB() {
        return b;
    }

    public void setB(double b) {
        this.b = b;
    }

    public abstract double calculate();
}

加法，除法运算类：

public class SumOperator extends Operator {

    @Override
    public double calculate() {
        return this.getA() + this.getB();
    }
}

public class DivisionOperator extends Operator  {
    @Override
    public double calculate(){
        if(this.getB() == 0){
            System.out.println("除数不能为0");
            return 0;
        }
        return this.getA() / this.getB();
    }
}

工厂类：

public class Factory {
    public static Operator getOperator(String op) {
        Operator operator = null;
        switch (op) {
            case "+":
                operator = new SumOperator();
                break;
            case "/":
                operator = new DivisionOperator();
                break;
        }
        return operator;
    }
}

客户端:

public class Client {
    public static void main(String[] args) {
        Operator operator = Factory.getOperator("+");
        operator.setA(1);
        operator.setB(2);
        System.out.println(operator.calculate());

        Operator op2 = Factory.getOperator("/");
        op2.setA(2);
        op2.setB(0);
        System.out.println(op2.calculate());
    }
}

# 优点缺点

优点：

  • 简化了客户端的代码，隐藏了实现细节。客户端无须直接创建对象，各自的责任更清晰

缺点：

  • 工厂类包含了所有类的创建细节，如果工厂类不能使用那么所有使用工厂创建的对象都无法使用
  • 增加新的具体实现类需要修改工厂类的逻辑，在一定程度上不利于维护和扩展

# 使用时机

  • 工厂类的逻辑不那么复杂
  • 客户端无须关注实现细节，只需要知道类对应的参数
